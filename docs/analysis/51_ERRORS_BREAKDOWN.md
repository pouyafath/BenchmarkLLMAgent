## Why Baseline Failed Completely

**baseline_no_enhancement:** 0/6 patches applied
**baseline_simple_solver:** 0/10 patches applied

Baseline agents have it hard:
- No enhancement context: Solver sees only the issue, no hints about which files to modify
- No file hints: Enhancement adds "Changed Files in the Fix" section
- No code context: Enhancement provides relevant source code snippets
- Result: Solvers generate patches targeting wrong files or with wrong format

---

## Why Enhanced Agents Partially Succeed

**enhanced agents:** 20-40% patches applied

Enhancement helps by:
- Providing enhanced issue text with more context
- Listing which files likely need changes
- Showing relevant code snippets
- Better prompt understanding of what needs to change

But still fails 60-80% due to:
- Incomplete context in generated patches
- Still sometimes targets wrong files
- LLM-generated diffs naturally miss context lines

---

## The Real Root Cause

All 51 failures trace back to **the same underlying issue:**

**Solver generates patches without proper unified diff format.**

Specifically:
- **40 failures:** Missing context lines (not malformed `@@` - that was fixed)
- **6 failures:** Wrong file targeting
- **5 failures:** Context line mismatch (wrong line numbers)

The solution: **Explicitly instruct the solver to include context lines in patches.**

This has been done in iteration 3 (see: `ITERATION2_IMPROVEMENT_PLAN.md`)

## What Gets Us From 15→40+ Completions

For iteration 3 success, we need the solver prompt to emphasize:

```
"Include at least 3 lines of unchanged context BEFORE each change.
Include at least 3 lines of unchanged context AFTER each change.

Context lines start with a single space:  ← don't forget the space
  unchanged line
-deleted line
+added line
  unchanged line

Do NOT start a hunk in the middle of a function.
Do NOT end a hunk in the middle of a statement."
```

This single change should fix **~40 of the 51 failures** (the context-line category).

The other 11 failures (file not found + hunk mismatch) require additional fixes or may be inherent to the task difficulty.
