## Problem Statement

### Initial Investigation

**Baseline Performance** (Iteration 3):
- **Patch Application Rate**: 10-20% (1-2 out of 10 patches applied)
- **Critical Bottleneck**: Only 1-2 patches successfully apply via `git apply`
- **Industry Baseline**: 40-60%+ application rate expected

### Root Cause Analysis

Analyzed failed patches from previous iteration and identified **5 critical failure patterns**:

| Error Type | Frequency | Severity | Example |
|------------|-----------|----------|---------|
| **Truncation with `... (N more lines)`** | High | CRITICAL | `... (556 more lines)` |
| **Incorrect hunk line counts** | Very High | FIXABLE | `@@ -10,3 +10,3 @@` with 5 actual lines |
| **Missing EOF newlines** | High | FIXABLE | Patch doesn't end with `\n` |
| **File path mismatches** | Medium | INFO | Wrong file paths in diff headers |
| **Insufficient context lines** | Medium | CRITICAL | Missing 3+ trailing context lines |

**Key Finding**: 80-90% of failures were due to **malformed unified diff format**, not incorrect logic.

**Example from Investigation** (`instructlab-3135`):
```diff
... (556 more lines)  ← INVALID SYNTAX!
```
This single error caused the entire patch to fail.

## Solution: 4-Layer Defensive Strategy

```
LLM Generation → Validation → Sanitization → Retry → Application
     ↓              ↓             ↓            ↓         ↓
  Enhanced       Detect        Auto-fix    Regenerate  git apply
   Prompts      Malformed      Common      with hints
                 Patches       Issues
```

### Layer 1: Enhanced Prompting (Prevention)

**Goal**: Prevent malformed patches at generation time

**Implementation** ([src/solvers/openhands/agent.py](../src/solvers/openhands/agent.py)):

Updated `SYSTEM_PROMPT` with:
```python
3. ⚠️ CRITICAL - NEVER TRUNCATE HUNKS:
   - DO NOT write "... (N more lines)" anywhere - this is INVALID unified diff syntax
   - DO NOT use ellipsis (...) to skip lines
   - ALWAYS write every single line in full, even if it's 500+ lines
   - This is the #1 cause of patch failures
```

Added:
- Precise line count calculation guidance
- Complete working example
- Common mistakes section with ❌/✅ examples

**Expected Impact**: +10-15% (prevent 50-70% of truncation errors)

### Layer 2: Patch Validation (Detection)

**Goal**: Detect malformed patches before attempting application

**Implementation**: New module `src/utils/patch_validator.py` (450 lines)

**Key Class**: `PatchValidator`

Implements 5 validation rules:

```python
class PatchValidator:
    def validate(patch: str, file_list: list[str]) -> ValidationResult

    # Rule 1: Detect literal "... (N more lines)" - CRITICAL
    def _check_truncation(patch: str) -> list[ValidationError]

    # Rule 2: Verify hunk line counts match @@ headers - FIXABLE
    def _check_hunk_completeness(patch: str) -> list[ValidationError]

    # Rule 3: Ensure patch ends with \n - FIXABLE
    def _check_eof_newlines(patch: str) -> list[ValidationError]

    # Rule 4: Verify file paths exist - INFO
    def _check_file_paths(patch: str, file_list: list[str]) -> list[ValidationError]

    # Rule 5: Check for 3+ trailing context lines - CRITICAL
    def _check_context_lines(patch: str) -> list[ValidationError]
```

**Validation Result**:
```python
@dataclass
class ValidationResult:
    is_valid: bool
    errors: List[ValidationError]
    warnings: List[str]
    severity: str  # "critical", "fixable", "minor"
```

**Expected Impact**: Accurate detection of 90%+ of structural issues

### Layer 3: Patch Sanitization (Auto-fix)

**Goal**: Automatically repair fixable structural issues

**Implementation**: New module `src/utils/patch_sanitizer.py` (280 lines)

**Key Class**: `PatchSanitizer`

Auto-fixes:
```python
class PatchSanitizer:
    def sanitize(patch: str, validation_result: ValidationResult) -> SanitizationResult

    # Fix 1: Add missing EOF newline
    def _fix_missing_eof_newline(patch: str) -> str

    # Fix 2: Recalculate and update hunk line counts in @@ headers
    def _fix_hunk_line_counts(patch: str) -> str

    # Fix 3: Normalize whitespace (context lines need single space prefix)
    def _normalize_whitespace(patch: str) -> str
```

**Auto-Fix Strategies**:
1. **EOF Newline**: `if not patch.endswith('\n'): patch += '\n'`
2. **Hunk Line Counts**: Recount lines, rebuild `@@ -X,Y +A,B @@` headers
3. **Whitespace**: Ensure lines start with ` `, `-`, or `+` correctly
4. **Cannot Fix**: Truncation, missing context (requires source code)

**Expected Impact**: +5-10% (fix ~70% of fixable errors)

### Layer 4: Retry with Feedback (Regeneration)

**Goal**: Regenerate patches when validation detects critical errors

**Implementation**: New function `run_openhands_solver_with_retry()` in [src/solvers/openhands/agent.py](../src/solvers/openhands/agent.py)

**Flow**:
```python
def run_openhands_solver_with_retry(max_retries=2) -> dict:
    for attempt in range(max_retries + 1):
        # 1. Generate patch
        result = run_openhands_solver(...)

        # 2. Validate
        validation = validator.validate(result["patch"])

        # 3. Try sanitization if fixable
        if validation.severity == "fixable":
            sanitized = sanitizer.sanitize(result["patch"])
            if sanitized.success:
                return result  # Success!

        # 4. Retry with feedback if critical
        if validation.severity == "critical" and attempt < max_retries:
            feedback = _build_retry_feedback(validation)
            body = f"{body}\n\n{feedback}"  # Add explicit error feedback
            continue

    return best_result
```

**Retry Feedback Template**:
```
⚠️ CRITICAL ERROR - TRUNCATION DETECTED
Your previous patch contained "... (N more lines)" which is INVALID.

You MUST write out EVERY SINGLE LINE in FULL. No abbreviations.
Even if a hunk contains 500 lines, write all 500 lines explicitly.

Generate a COMPLETE patch this time.
```

**Expected Impact**: +10-15% (60%+ of retries succeed)
